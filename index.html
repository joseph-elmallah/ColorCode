Color Code

by Joseph Mallah

Color Code is a jQuery plug-in that transform numerical tags into color scale. The advantage of color coding is to facilitate the visualization of the data and provide an instant understanding of the values.

The basic usage of Color Code is to parse specific div (or any other tag) and color their background using a color gradient depending on the contained value inside the div.

Requirements

Color code require the jQuery library version 1.9 and up. In addition to the jQuery library, color code requires the rainbowvis plug-in to generate the color gradients.

Installation

Load the required JavaScripts libraries in the proper order, then load the color code library

  1. jQuery library
  2. rainbowvis plug-in
  3. colorCode plug-in

Usage

To use the plug-in simply call the plug-in on your elements. The value parsed should be an INTEGER otherwise the plug-in will throw an error.

To specify more settings, pass them to the function in the options parameter:

Options

  • clearContent [true/false]:
    Whether to remove the number inside the div after coloring the background or leave the content untouched.
    Default: false
  • colors [colour1, colour2 [,colourN]]:
    An array of colors to form the gradient. A minimum of two colors are required. Colors can be in the form 'red', 'ff0000', or '#ff0000'.
    Default: ["green", "yellow", "red"]
  • proportional [true/false]:
    Whether colors are proportional to the numbers. If set to true, an interval of [0, 5, 100] will have a 100 gradient colors, meaning that the number 0 will have the start color and the 100 will have the end color, but the number 5 color will be closer to start color since 5 is closer to 0 than to 100. Otherwise, if set to false the number 5 will have the central color between the starting and ending colors.
    Default: true
  • useDataTag [string/null]:
    Whether to use the HTML content of the tag or the data contained in a HTML5 data attribute. If the value is null, the HTML content of the tag will be parsed. Otherwise if a string is specified than the content of the data attribute of this string will be used.
    Default: null
